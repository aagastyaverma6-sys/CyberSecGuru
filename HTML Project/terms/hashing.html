Hashing

Category: Integrity Difficulty: Intermediate Related: Encryption, CVE

Definition

Hashing is a one-way mathematical transformation that converts input data into a fixed-length string, used to verify integrity.

Why it matters: Hashes ensure that data has not been altered or tampered with.

Examples

  • SHA-256 for blockchain transactions.
  • MD5 checksums for file downloads.
  • Password storage with salted hashes.

Best practices

  • Use modern algorithms (SHA-2, SHA-3).
  • Never store plain hashes of passwords—always salt them.
  • Verify file integrity after transfers.
