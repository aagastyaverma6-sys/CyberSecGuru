Cross-Site Request Forgery (CSRF)

Category: Web Security Difficulty: Intermediate Related: SQL Injection, Authentication

Definition

CSRF is an attack that tricks a user’s browser into executing unwanted actions on a web application where they are authenticated.

Why it matters: CSRF can lead to unauthorized transactions or data changes without the user’s knowledge.

Examples

  • Forcing a logged-in user to transfer funds.
  • Changing account settings via hidden requests.

Best practices

  • Use anti-CSRF tokens.
  • Implement SameSite cookie attributes.
  • Require re-authentication for sensitive actions.
